Example: Replace a base element with the domain extensions using RNG

In this scenario, an information architect wants to remove the <ph> element but allow the extensions of <ph> that exist in the highlight, programming, software, and user interface domains.

  1. She opens the document-type shell for topic in an XML editor, and then modifies the "MODULE INCLUSIONS" division to exclude <ph>: 

    <div>
  <a:documentation>MODULE INCLUSIONS</a:documentation>    
  <include href="urn:oasis:names:tc:dita:rng:topicMod.rng:2.0">
    <define name="ph.element">
      <notAllowed/>
    </define>
  </include>
  ...
</div>
  2. She makes similar changes to all the other document-type shells in which she wants <ph> to not be available
