LwDITA universal attribute group

The universal attribute group defines a set of common attributes that are available on most XDITA and HDITA components, and have limited equivalence in MDITA components. The universal attribute group includes all attributes from the ID, metadata, and localization attribute groups, plus the @class and @outputclass attributes.

Common attribute groups

The following groups are referenced in this specification, and are also used in grammar files when defining attributes for components.

Universal attributes
Includes @class and @outputclass, along with every attribute in the ID, Localization, and Metadata attribute groups.
ID attributes
Includes attributes that enable the naming and referencing of elements in topics and maps: @conref and @id.
Localization attributes
Includes attributes that are related to translation and localization: @dir, @translate, and @language.
Metadata attribute
Includes the generic conditional processing attribute @props.

@outputclass
@outputclass in XDITA, @class in HDITA, and {.outputclass) in MDITA following the header_attributes Pandoc extension in MDITA)
Names a role that the component is playing. The role must be consistent with the basic semantic and expectations for the element. In particular, the @outputclass XDITA attribute can be used for styling during output processing; HTML output will typically preserve @outputclass for CSS processing.
Authors combining XDITA (or DITA) and HDITA as formats for content creation in a single project or repository MUST NOT confuse the HDITA @class attribute with the similarly named @class in XDITA, which does not have a direct equivalent in HDITA.
In, MDITA the @outputclass attribute is only available in the <topic> and <section> components.
